# Handoff Guides

These guides define how to create and maintain concise, reliable handoff docs for Gantryland work sessions.

## Guides

- [`docs/handoff/quick-handoff.md`](quick-handoff.md) - Current release-session handoff snapshot.
- [`docs/handoff/authoring.md`](authoring.md) - Rules and template for writing handoff docs with an LLM.

## Scope boundaries

- `quick-handoff.md` is a point-in-time operational status document.
- `authoring.md` is a reusable standard for creating or updating handoff docs.

## Recommended usage order

1. Start with `quick-handoff.md` for immediate session context.
2. Use `authoring.md` when creating or refreshing handoff content.
